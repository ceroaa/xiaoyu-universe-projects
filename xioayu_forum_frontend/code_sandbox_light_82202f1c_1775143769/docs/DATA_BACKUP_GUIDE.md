# 📦 資料備份與管理指南

**版本**: 1.0  
**更新時間**: 2026-03-09

---

## 🎯 概述

小雨論壇資料管理系統提供完整的本地資料備份、匯出、匯入功能。所有用戶資料均存儲在瀏覽器的 `localStorage` 中，您可以輕鬆備份並在不同設備間遷移資料。

---

## 📂 包含的資料類型

### 用戶資料
- ✅ 用戶個人資訊（姓名、ID、頭像）
- ✅ XYC 餘額
- ✅ 登入獎勵記錄（連續登入天數、最後領取時間）
- ✅ 心情動態（所有發布的動態與點讚記錄）

### 系統設定
- ✅ 主題偏好（淺色/深色模式）
- ✅ 語言設定
- ✅ 個人化設定

### 活動記錄
- ✅ 彩蛋獎勵歷史
- ✅ 兌換記錄
- ✅ AI 聊天訊息
- ✅ 自動備份記錄

### 2. 匯出資料

#### 📥 匯出全部資料
**用途**: 完整備份所有論壇資料  
**文件格式**: JSON  
**文件名**: `xiaoyu-forum-backup-{timestamp}.json`

**包含內容**:
```json
{
  "version": "1.0",
  "timestamp": "2026-03-09T15:30:00.000Z",
  "data": {
    "currentUser": {...},
    "xycBalance": "5000",
    "xiaoyuForum_moodPosts": [...],
    "lastLoginRewardTime": "1234567890",
    ...
  },
  "stats": {
    "totalSize": 12345,
    "totalKeys": 15,
    ...
  }
}
```

#### 📥 僅匯出用戶資料
**用途**: 快速備份核心用戶資料  
**文件格式**: JSON  
**文件名**: `xiaoyu-forum-user-{timestamp}.json`

**包含內容**:
- `currentUser` - 用戶資訊
- `xycBalance` - XYC 餘額
- `agentIdDisplay` - 代理ID
- `lastLoginRewardTime` - 最後獎勵時間
- `lastLoginDate` - 最後登入日期
- `loginStreak` - 連續登入天數
- `xiaoyuForum_moodPosts` - 心情動態

#### 👁️ 預覽資料
**用途**: 在匯出前查看資料內容  
**顯示**: JSON 格式化預覽  
**位置**: 頁面內即時顯示

---

### 3. 匯入資料

#### 📤 拖放上傳
1. 將之前匯出的 `.json` 文件拖放到「匯入資料」區域
2. 系統自動讀取並確認
3. 點擊「確定」完成匯入

#### 📤 文件選擇
1. 點擊「匯入資料」區域
2. 選擇 `.json` 文件
3. 確認匯入

#### ⚠️ 匯入注意事項
- 匯入**將覆蓋**現有所有資料
- 系統會在匯入前自動創建臨時備份（存於 `sessionStorage`）
- 匯入失敗時會自動恢復到匯入前狀態
- 匯入成功後頁面將自動重新載入

---

### 4. 自動備份

#### 🤖 自動備份機制
- **頻率**: 每 **7 天**自動創建備份
- **保留數量**: 最多 **3 個**備份
- **存儲位置**: `localStorage` 的 `backup_history` 鍵

#### 📜 備份歷史
每個備份包含：
- 📅 **備份時間**: 精確到秒
- 💾 **文件大小**: 以 KB 顯示
- 🔧 **操作按鈕**:
  - **恢復**: 將備份資料還原到系統
  - **刪除**: 移除備份記錄

#### 🔄 手動觸發自動備份
訪問資料管理頁面時，系統會檢查：
- 如果距離上次自動備份 ≥ 7 天 → 自動創建新備份
- 否則 → 使用現有備份

---

### 5. 危險操作

#### 🗑️ 清除所有資料
**效果**: 永久刪除 `localStorage` 和 `sessionStorage` 中的所有資料

**安全措施**:
1. 第一次確認對話框
2. 第二次確認（需輸入「確認刪除」）
3. 完成後自動重新載入

#### 🔄 重置為預設狀態
**效果**: 清除用戶資料，保留基本結構

**重置後保留**:
- 主題設定 = `light`
- XYC 餘額 = `0`
- 登入狀態 = `false`

## 📝 使用場景

### 場景 1: 跨設備遷移
**問題**: 想在新電腦上繼續使用論壇  
**解決方案**:
1. 在舊設備上 **「匯出全部資料」**
2. 將 JSON 文件傳輸到新設備（雲端硬碟、郵件、USB 等）
3. 在新設備上訪問 `data-manager.html`
4. **「匯入資料」** 上傳 JSON 文件

### 場景 2: 定期備份
**問題**: 擔心資料丟失  
**解決方案**:
- 每週訪問 `data-manager.html` 查看自動備份狀態
- 或手動 **「匯出全部資料」** 保存到雲端

### 場景 3: 測試與實驗
**問題**: 想測試功能但不想影響真實資料  
**解決方案**:
1. **「匯出全部資料」** 創建備份
2. 進行測試（發布動態、兌換等）
3. 測試完成後 **「匯入資料」** 恢復到測試前

### 場景 4: 重新開始
**問題**: 想清空所有資料重新註冊  
**解決方案**:
- 使用 **「清除所有資料」** 或 **「重置為預設狀態」**

## 🛡️ 資料安全

### ✅ 已實施的安全措施
1. **本地存儲**: 資料僅存於瀏覽器 localStorage，不上傳伺服器
2. **匯入保護**: 雙重確認機制
3. **臨時備份**: 匯入前自動備份至 sessionStorage
4. **自動恢復**: 匯入失敗時自動恢復
5. **版本標記**: 匯出文件包含版本號，便於未來相容性

### ⚠️ 已知限制
1. **前端存儲**: 清除瀏覽器資料會刪除所有論壇資料
2. **無加密**: JSON 文件未加密，請妥善保管
3. **大小限制**: localStorage 約 5-10 MB（各瀏覽器不同）

## 🔧 技術實現

### 核心文件
- **`data-manager.html`** (11.3 KB) - 資料管理頁面
- **`js/data-manager.js`** (13.8 KB) - 資料管理邏輯

### 核心類別
```javascript
class DataManager {
  updateStats()         // 更新資料統計
  exportAllData()       // 匯出全部資料
  exportUserData()      // 匯出用戶資料
  previewData()         // 預覽資料
  importData()          // 匯入資料
  saveToBackupHistory() // 保存備份歷史
  loadBackupHistory()   // 載入備份歷史
  checkAutoBackup()     // 檢查自動備份
  clearAllData()        // 清除所有資料
  resetToDefault()      // 重置為預設
}
```

### localStorage 鍵列表
| 鍵名 | 類型 | 說明 |
|-----|------|------|
| `currentUser` | Object | 用戶資訊（name, id, type, avatar） |
| `xycBalance` | String | XYC 餘額 |

| `lastLoginRewardTime` | String | 最後獎勵時間戳 |
| `lastLoginDate` | String | 最後登入日期 |

| `xiaoyuForum_moodPosts` | Array | 心情動態列表 |

## 🎯 最佳實踐

### 建議備份頻率
- **一般用戶**: 每週一次
- **活躍用戶**: 每天一次
- **測試環境**: 每次測試前

### 備份文件管理
1. **命名規範**: 保留文件原始時間戳名稱
2. **存儲位置**: 雲端硬碟（Google Drive、Dropbox、OneDrive）
3. **保留數量**: 至少 3 個最近備份
4. **定期清理**: 刪除 30 天以上的舊備份

### 跨設備同步
- 使用雲端硬碟自動同步備份文件
- 建立定期備份提醒（手機日曆）

## ❓ 常見問題

### Q1: 匯出的 JSON 文件可以手動編輯嗎？
**A**: 可以，但需確保 JSON 格式正確。建議使用 JSON 驗證工具檢查。

### Q2: 自動備份會影響性能嗎？
**A**: 不會。自動備份僅在訪問資料管理頁面時檢查，不會在後台運行。

### Q3: 可以在不同瀏覽器間共享資料嗎？
**A**: 可以。匯出 → 傳輸 → 匯入即可。

### Q4: 清除所有資料後可以恢復嗎？
**A**: 無法恢復。請務必先匯出備份！

### Q5: 匯入舊版本的備份會有問題嗎？
**A**: 系統會嘗試相容舊版本，但建議使用最新版本匯出的備份。
